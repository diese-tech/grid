# Alpha Build Spec

## Purpose

This doc defines what the first real version of The Grid must prove. Alpha is not the final universe. Alpha is the first connected sector that shows the vibe, traversal, Lab, and node system actually work.

---

# Q/A Summary

1. Alpha must prove the vibe works, then that the node system works, and most importantly that traversal and The Lab feel real.

2. Day-one pride requires a homepage, The Lab, and active nodes.

3. Alpha should not include deep expansion, excessive future sectors, or every cool idea.

4. First represented projects should include GodForge and SwiftDispatch as staple anchor nodes, with StickyRiceKitchen and HersonBot as WIP nodes.

5. Project data should be real where possible. Mock/static data is acceptable only when needed to test visualizations that cannot be integrated yet.

6. The intro should be fully polished in alpha. The edges of the world can be rough, but the intro should not feel rough.

7. Sound should exist in alpha.

8. Hidden content should be limited. More can be added later.

9. The minimum Lab experience is connectivity, awareness, and strong vibes.

10. The minimum homepage/transit experience is active traversal, functioning menus, and functioning nodes.

11. Alpha would feel too small if it loses the movement idea or settles for what is easy.

12. Alpha would feel overbuilt if it contains too many construction zones or too many areas obviously primed for later.

13. The first public CTA is still unresolved, but possible directions include “Enter The Grid,” “Welcome to The Grid,” “Become Unbound,” “Challenge Yourself,” or a more human contact line like “We can’t talk here, email me.”

14. Alpha is for all key audiences: the creator, collaborators, friends/community, builders, future clients, and Twitch/community.

15. Alpha can launch when the core is built.

---

# Alpha Goal

Alpha must prove:

- The Grid has a memorable vibe
- traversal feels different from a normal website
- The Lab feels like a real place
- nodes feel alive and meaningful
- active projects ground the experience in reality
- the intro creates a strong first impression

Alpha is successful if someone thinks:

> “I need something like that.”

---

# Required Alpha Experience

## 1. Polished Intro

The intro must feel intentional and finished.

It should communicate:

- system wake-up
- arrival into The Grid
- operational atmosphere
- polished transition into the homepage/transit layer

The intro is not allowed to feel like a placeholder.

---

## 2. Homepage / Transit Layer

The homepage must function as:

- transit hub
- orientation layer
- navigation shell
- atmospheric entry point

Minimum requirements:

- obvious navigation options
- active traversal behavior
- station/transit atmosphere
- node map or route map presence
- small welcome/operator message
- functioning movement into The Lab

The homepage should not try to explain everything.

---

## 3. The Lab

The Lab is the emotional core of alpha.

Minimum requirements:

- strong atmosphere
- visible project/node presence
- operator/workstation feeling
- active systems or simulated active-state behavior
- connectivity between nodes
- awareness that systems are linked

The Lab should feel alive even if not every system is fully integrated yet.

---

## 4. Active Node System

Alpha must include active nodes.

Required anchor nodes:

- GodForge
- SwiftDispatch

Required WIP nodes:

- StickyRiceKitchen
- HersonBot

Nodes should show at minimum:

- name
- status
- importance level
- small preview
- project type
- basic relationship to other nodes

---

# Data Strategy

Use real data where possible.

Priority real integrations:

1. GitHub repository metadata
2. recent commit/activity signals
3. project links
4. deployment/status data if simple enough

Static or mock data is allowed only when:

- the integration is not ready
- the visualization needs testing
- live data would create unnecessary scope creep

Mocked data must not pretend to be live if it is not live.

---

# Sound Requirements

Sound should exist in alpha.

Minimum sound direction:

- subtle ambience
- optional mute
- soft interaction feedback
- restrained environmental sound

Sound should support immersion without annoying visitors.

---

# Hidden Content Rules

Alpha should contain a small amount of hidden content.

Good alpha hidden content:

- one or two subtle anomalies
- small hidden joke
- light inactive-node behavior
- minor environmental change after idle time

Avoid:

- full ARG systems
- dozens of secrets
- deep unlock trees
- lore-heavy hidden paths

---

# Alpha Non-Goals

Do not build yet:

- full universe expansion
- large number of sectors
- deep lore systems
- complex account/user system
- advanced AI assistant
- heavy homelab telemetry
- full Discord/Twitch/Spotify integration suite
- many construction zones
- excessive hidden mechanics
- complete project archive

The goal is not scale.

The goal is proof of identity.

---

# Scope Guardrails

## Too Small

Alpha is too small if:

- it feels like a normal homepage
- traversal is missing
- The Lab feels like just another page
- nodes are only cards
- movement is sacrificed for ease
- the intro is weak or absent

## Overbuilt

Alpha is overbuilt if:

- too many sectors exist
- too many areas are placeholders
- construction zones dominate
- hidden mechanics distract from the core
- every reference idea gets included
- the system becomes hard to finish

---

# CTA Direction

The CTA is unresolved and should be explored carefully.

Possible CTA language:

- Enter The Grid
- Welcome to The Grid
- Become Unbound
- Challenge Yourself
- We can’t talk here, email me

CTA should not feel like corporate conversion copy.

The CTA should feel like an invitation into the world.

---

# Launch Rule

Alpha can launch when the core is built.

Core means:

- polished intro
- functional homepage/transit layer
- memorable Lab
- active node system
- first project nodes
- basic sound layer
- enough traversal to feel spatial

Anything beyond that is optional.

---

# Final Alpha Definition

GRID // ALPHA is:

> a polished first connected sector proving that The Grid’s traversal, Lab, node system, atmosphere, and active project identity work together as one memorable experience
