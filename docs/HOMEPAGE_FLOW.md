# Homepage Flow

## Purpose

This doc defines the first real experience of The Grid. The homepage is not a normal landing page. It is the threshold: the place where the user boots into The Grid, gets oriented, and chooses where to go next.

---

# Q/A Summary

1. The first emotions should be intrigue and curiosity.

2. The intro should happen before the homepage, like the user is booting up The Grid.

3. The first scene should feel like a station platform with obvious navigation options.

4. Users should understand enough to navigate, but not everything should be explained immediately.

5. Initial attention should go to environmental movement, node maps, and operator messages.

6. The homepage is an orientation layer, transit hub, and navigation shell.

7. The homepage should contain some meaningful introductory content, including a small “welcome to The Grid” style message and some entry-level context about Diese and the idea ecosystem.

8. Alpha should start with the first few active GitHub/projects already identified in the roadmap and Lab concept.

9. Some sectors should be visible immediately. Others can require curiosity and interaction, but not time-locking.

10. If someone idles on the homepage, the environment should slowly become stranger: distant train horns, Station 8 anomalies, DVD-menu-like loops, classic game-menu ambient behavior, subtle changes, and environmental unease.

11. The user should learn about Diese through a mix of direct intro and exploration.

12. Stable active projects should be visible and present on the node map. WIP projects can be mixed into deeper or stranger sectors depending on their maturity.

13. Homepage transitions should be hybrid: seamless, environmental, map-like, and scene-aware.

14. Visitors should remember The Lab, the node maps, and the transitions after leaving the homepage.

15. The biggest risk is idea-clash: the homepage trying to express too many concepts at once.

---

# Core Homepage Philosophy

The homepage is not the destination.

The homepage is the threshold.

It should function as:

- orientation layer
- transit hub
- navigation shell
- atmospheric introduction
- system entry point

The homepage should not attempt to explain all of The Grid.

It should create enough clarity to move forward and enough mystery to keep exploring.

---

# Boot Sequence

The intro should feel like booting into The Grid.

It should establish:

- system wake-up
- operational tone
- arrival into a place
- transition from outside world into Grid environment

Avoid fake hacker boot screens, fake BIOS spam, or long forced animations.

The boot should lead naturally into the station/transit layer.

---

# Homepage Spatial Role

The homepage should feel like a quiet transit station.

Emotional references:

- eerie Japanese train station
- calm transit platform
- quiet operational hub
- liminal waiting space
- destinations implied beyond the visible area

The station idea works because it naturally suggests:

- movement
- destinations
- systems
- routes
- waiting
- unseen sectors
- connected worlds

---

# First Visible Scene

The first real scene should include:

- station/platform atmosphere
- obvious navigation options
- subtle environmental movement
- node map or route map presence
- operator message or welcome signal

The first impression should be:

> “What is this place, and where can I go?”

---

# Homepage Content Rules

The homepage should include some content, but it should not become a normal bio page.

Allowed content:

- small welcome message
- short operator intro
- visible stable project nodes
- current activity/status hints
- route/node map
- invitation to enter The Lab

Avoid:

- long paragraphs
- resume structure
- excessive explanation
- trying to show every project at once

---

# Visible vs Hidden Sectors

## Visible immediately

Stable and active systems:

- major projects
- The Lab
- active GitHub/project nodes
- primary navigation routes

## Discoverable

Curious users can uncover:

- WIP nodes
- inactive sectors
- memory fragments
- hidden jokes
- deeper routes
- construction zones

Unlocking should be curiosity-based, not time-locked.

---

# Idle Behavior

If a user idles on the homepage, the environment should slowly become stranger.

Possible behaviors:

- distant train horn
- lighting shifts
- station announcement distortion
- subtle route-map change
- inactive node pulse
- environmental loop variation
- anomalous movement in the background
- DVD-menu / classic-game idle behavior

This should be subtle and atmospheric, not disruptive.

---

# Transition Philosophy

Homepage transitions should feel like moving between sectors.

Preferred transition types:

- environmental pans
- map/deployment transitions
- perspective shifts
- seamless sector movement
- scene-aware cuts

Avoid transitions that feel like normal page loads.

---

# Homepage Guardrail

The homepage must unify the ideas instead of showcasing them all equally.

The fix:

> Homepage = transit + orientation.

Let The Lab carry density, chaos, experimentation, and deeper identity.

---

# Homepage Success Criteria

The homepage succeeds if visitors remember:

- the station/transit feeling
- the node map
- the boot into The Grid
- the invitation toward The Lab
- the feeling that the world extends beyond the screen
