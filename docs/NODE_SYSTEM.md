# Node System

## Purpose

This doc defines how nodes work inside The Grid. Nodes are not just project cards. They are the living map of projects, systems, memories, aspirations, people, and ideas that make the environment feel connected.

---

# Q/A Summary

1. Nodes represent active projects, GitHub repositories, ideas, memories, aspirations, people, and active operations.

2. Nodes should vary in size and importance. Some should become anchor nodes that ground the experience in reality.

3. Anchor nodes represent major projects or systems that created future pathways. For example, GodForge paved the way for the OCR stat-tracking bot, ambitious websites, and Smite 2 production tooling.

4. Node states can include active, unstable, archived, abandoned, hidden, redacted, under construction, corrupted, live, and experimental.

5. Active nodes can have a faint hum and greenish glow. Unstable nodes can blink or shake at random intervals. Archived nodes should be dimmer and less vibrant. Abandoned nodes can feel cracked, broken, or glass-like. Hidden/redacted nodes can feel SCP-style. Under-construction nodes can feel Boneworks-esque. Corrupted nodes can use reddish glow and orange rays from visible cracks. Experimental nodes can use cool blue/cyan hues.

6. Connections should feel like Astroneer tethers: glowing, slightly unstable, flickering, and lightly pulsing.

7. The map should feel like a hybrid of neural, machine-like, and organic structure.

8. Before clicking, a node should show a tiny preview with system health, name, and importance level.

9. Clicking a normal active node should open or update an info panel. Clicking a themed, inactive, hidden, or scene-specific node can trigger an environmental shift.

10. Node brightness should be context-aware. In rough outer sectors, construction nodes may be brighter than active nodes because they belong to that environment.

11. Nodes can physically appear in different ways depending on location: floating holograms, wall projections, suspended systems, planning-table points, or projected architecture.

12. Inactive and dead nodes should feel like the last lightning bug in the night sky.

13. Nodes should update live over time through commits, logs, deployment status, project activity, stream activity, or system uptime.

14. Hidden nodes should be discoverable through exploration, attention, and curiosity.

15. Deep nodes should feel like another realm. These are emotionally significant projects, personal businesses, identity-heavy systems, or major internal projects like The Grid itself.

16. Zooming far out can trigger strange behavior, like a robotic claw from the Lab workstation pulling the camera back after too long.

17. Zooming very close can reveal subtle strangeness, like tiny visual anomalies or eye-like movement at the edge of perception.

18. A strong click interaction could include a light hum, pulse, and water-ripple effect.

# Core Node Philosophy

The node system is the map of The Grid’s living ecosystem.

It should communicate:

- work
- memory
- ambition
- project lineage
- active systems
- unfinished pathways
- emotional significance

Nodes should feel like pieces of a larger machine and pieces of a larger personal story.

---

# Anchor Nodes

Anchor nodes are the most important nodes in the map.

They ground The Grid in real work.

Anchor nodes can represent:

- major projects
- projects with real users
- projects that created future branches
- personally meaningful systems
- businesses or serious ventures

Anchor nodes should be visually larger, more stable, and more connected.

Examples:

- GodForge
- SwiftDispatch
- ThreeTails booking / scheduling
- The Grid itself

---

# Node States

## Active

Represents a live or maintained system.

Visual direction:

- faint hum
- subtle greenish glow
- steady pulse
- stable tether behavior

## Unstable

Represents an active but unreliable, experimental, or uncertain system.

Visual direction:

- irregular blinking
- shaky glow
- random flickers
- uneven pulse timing

## Archived

Represents a past system or inactive memory.

Visual direction:

- dimmed
- lower saturation
- soft presence
- visible but quiet

## Abandoned

Represents a dead or failed path.

Visual direction:

- cracked glass
- broken screens
- severed tethers
- visual damage
- low remaining glow

## Hidden / Redacted

Represents intentionally obscured or discoverable systems.

Visual direction:

- redacted labels
- partial reveal
- SCP-style secrecy
- obscured metadata
- subtle signal leakage

## Under Construction

Represents active building, future areas, or unfinished systems.

Visual direction:

- Boneworks-esque structure
- scaffolding
- exposed framework
- brighter in rough/construction sectors
- incomplete geometry

## Corrupted

Represents broken, unstable, or dangerous systems.

Visual direction:

- red/orange glow
- visible cracks
- rays or heat leaking from damage
- unstable pulse

## Experimental

Represents active exploration and uncertain direction.

Visual direction:

- blue/cyan hues
- cool glow
- less stable than active nodes
- exploratory labeling

---

# Tether System

Connections between nodes should feel inspired by Astroneer tethers.

Tethers should feel:

- glowing
- lightly unstable
- alive
- pulsing
- physical enough to understand
- abstract enough to remain atmospheric

Tethers can communicate:

- dependency
- inspiration
- project lineage
- system relationship
- dead paths
- future expansion

Dead tethers are important.

They imply:

- abandoned ideas
- missing systems
- broken pathways
- future possibilities

---

# Pre-Click Information

Before clicking, a node can show:

- name
- system health
- importance level
- status
- small preview
- category

This keeps the map readable without overwhelming the user.

---

# Click Behavior

## Normal Active Node

Clicking should:

- pulse the node
- create a light hum
- trigger a ripple or signal effect
- update the active info panel
- show current status and project details

## Themed / Scene Node

Clicking can:

- trigger an environmental shift
- transition to another sector
- reveal a deeper system
- change the surrounding atmosphere

Scene-specific nodes should become brighter when they belong to that environment.

---

# Physical Representation

Nodes can appear as:

- floating holograms
- projected architecture
- wall projections
- table-map points
- suspended systems
- physical modules
- digital fragments

The representation should match the sector.

---

# Deep Nodes

Deep nodes represent emotionally significant systems.

They may include:

- personal businesses
- identity-heavy projects
- major life systems
- The Grid itself
- personal creative ambitions

Deep nodes should feel like another realm.

They should be harder to reach, stranger, and more emotionally charged.

---

# Zoom Behavior

## Zoomed Out

Zooming far out should imply scale.

If the user stays too far out too long, a strange event can occur.

Example:

- a robotic claw from the Lab workstation pulls the camera back into the working range

## Zoomed In

Zooming very close can reveal subtle anomalies.

Examples:

- tiny eye-like movement
- micro-glitches
- ripple artifacts
- background strangeness
- hidden texture details

These should be subtle and rare.

---

# Live Update Philosophy

Nodes should eventually update live.

Possible update sources:

- GitHub commits
- project logs
- Vercel deployments
- Twitch status
- Discord activity
- system uptime
- homelab telemetry

Live updates make the map feel alive and grounded in real systems.

---

# Final Node System Goal

The node system should feel like:

> a living tech-tree of projects, memories, systems, failures, aspirations, and active operations connected by unstable tethers inside an evolving operational dreamspace
