# Reference Collection Guide

## Purpose

This doc explains how references should be collected for The Grid. The goal is not to copy visuals directly. The goal is to identify emotional patterns, environmental logic, interaction ideas, and atmosphere.

---

# Q/A Summary

1. The references folder can include screenshots, videos, UI frames, game menus, movie stills, sound references, architecture references, objects, typography, and general vibe captures.

2. References should be tiered by strength. Higher-tier references mean “more of this if it fits,” not “copy this directly.”

3. Anything that breaks the emotional direction of The Grid should be rejected unless explicitly marked with a note explaining why it belongs.

4. Boneworks-like atmosphere and conceptual space logic are part of The Grid’s visual DNA.

5. The Grid should feel ambitious, atmospheric, immersive, and spatial. Simplicity without atmosphere is usually a warning sign.

---

# Core Rule

Collect references for:

- feeling
- atmosphere
- interaction
- spatiality
- emotional memory
- environmental storytelling

Not just aesthetics.

# Tiering System

## Tier 1 — Core DNA

References that strongly match The Grid.

Examples:

- Deep Stone Crypt
- Kamino
- Portal 2
- spatial PS2 menu systems
- Boneworks atmosphere

## Tier 2 — Supporting Inspiration

References that support pieces of the experience.

Examples:

- interface ideas
- motion pacing
- sound design
- typography inspiration
- object design

## Tier 3 — Experimental

References that might not fully fit but contain:

- interesting emotion
- strange atmosphere
- interaction ideas
- unusual environmental storytelling

These require notes explaining why they belong.

---

# Immediate Rejection Signals

Reject references that feel:

- generic
- template-driven
- crypto-themed
- AI-slop-like
- fake hacker aesthetic
- overly neon cyberpunk
- visually noisy
- corporate SaaS
- trend-chasing

---

# Reference Evaluation Questions

When adding a reference, ask:

- Does this feel inhabited?
- Does this create atmosphere?
- Does this imply systems beneath the surface?
- Does this feel memorable?
- Does this support spatial immersion?
- Would this still feel cool without flashy effects?

If the answer is no, it probably does not belong.

---

# Final Reminder

The Grid is not trying to become:

- a generic futuristic UI
- a dashboard template
- a fake operating system
- a cyberpunk clone

The Grid should feel like:

> a remembered operational dreamspace that somehow already existed in the user's memory
