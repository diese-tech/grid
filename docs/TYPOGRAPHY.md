# Typography

## Purpose

This doc keeps the words inside The Grid from feeling random. The typography should carry both the clean sci-fi operating system side and the personal journal side without turning corporate or unreadable.

---

# Q/A Summary

1. The Grid should speak through a hybrid of clean sci-fi operating system text and editorial/personal journal text.

2. The main readable font should land between modern neutral and technical. It should feel clean, capable, and slightly system-like without becoming cold or robotic.

3. Text should only become dense inside The Lab, workshop pages, archives, and rough experimental zones. Outer/public areas should stay cleaner.

4. Big headings should carry meaning. They can feel cinematic and old game-menu inspired, potentially animated or spatially staged.

5. Tiny UI labels should feel like a mix of old OS labeling and game HUD language.

---

# Typography Personality

The Grid uses typography to balance two voices:

- the operational shell
- the human operator

The operational shell is clean, technical, structured, and restrained.

The human operator is reflective, funny, ambitious, and occasionally chaotic.

Both should coexist.

---

# Font Roles

## Primary UI Font

Used for:

- navigation
- body text
- panels
- project descriptions
- readable content

Should feel:

- modern
- neutral
- technical
- legible
- durable

Avoid fonts that feel too friendly, too corporate, or too futuristic.

## Display / Heading Font

Used for:

- large section headers
- cinematic moments
- sector titles
- intro states
- major scene transitions

Should feel:

- cinematic
- slightly game-menu inspired
- meaningful
- atmospheric

Headings should not just be large. They should feel like moments.

## Micro-label Font

Used for:

- tags
- status labels
- metadata
- node details
- tiny panel language

Should feel like:

- old OS labels
- game HUD text
- command-center metadata

This is where nostalgia can quietly live.

---

# Text Density Rules

## Low Density Areas

Use lower density in:

- homepage shell
- outer Grid areas
- first impressions
- top-level navigation
- clean sector entrances

These areas should be scannable and atmospheric.

## High Density Areas

Use higher density in:

- The Lab
- workshop pages
- project details
- archives
- operator notes
- raw experiments

Density should signal depth, not clutter.

---

# Heading Rules

Headings should:

- carry emotional weight
- establish place
- support spatial navigation
- feel integrated into the environment

Allowed heading behavior:

- subtle animation
- cinematic entrance
- layered positioning
- old menu-inspired staging
- environmental reveal

Avoid:

- generic huge SaaS headlines
- random cyberpunk lettering
- unreadable novelty fonts
- empty motivational copy

---

# Labeling Rules

Labels should be useful but atmospheric.

Good labels:

- NODE STATUS
- ACTIVE SIGNAL
- ARCHIVED FRAGMENT
- LAB SECTOR
- BUILD STATE
- RAW SYSTEM

Bad labels:

- fake military commands
- over-the-top hacker phrases
- meaningless sci-fi jargon

---

# Human Text

Human text can appear as:

- notes
- captions
- jokes
- failed idea labels
- commentary
- quick observations
- project scars

This text should sound like Diese, not a brand deck.

---

# Typography Guardrail

If the typography looks like a generic startup site, it is wrong.

If it looks like a fake hacker dashboard, it is wrong.

The correct direction is:

> clean system language with human fingerprints
