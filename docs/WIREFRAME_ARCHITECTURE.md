# Wireframe Architecture

## Purpose

This document defines the high-level structural layout rules for The Grid. These wireframes are not focused on visual polish yet. They define movement, composition, exploration flow, environmental hierarchy, and how users physically experience the world.

The primary design challenge is not aesthetics.

The primary design challenge is:

> maintaining coherent exploration flow.

---

# Core Structural Philosophy

The Grid should feel traversed, not browsed.

Users should learn the world:

- spatially
- emotionally
- environmentally
- through recurring routes
- through remembered sectors
- through node relationships

The experience should avoid feeling like:

- a normal website
- dashboard software
- disconnected pages
- infinite scrolling content

---

# Core Wireframe Rules

## Rule 1 — Environment First

The environment is the primary interface.

UI exists inside or around the environment.

The environment should communicate:

- mood
- navigation
- hierarchy
- activity
- sector identity
- emotional state

---

## Rule 2 — Traversal Builds World

Worldbuilding should happen through:

- movement
- transitions
- node relationships
- environmental continuity
- recurring spaces
- route discovery

Scrolling is reserved for:

- dossiers
- detailed project info
- operational logs
- deeper records

---

## Rule 3 — Sector Layouts Can Change

The Grid should not use one rigid webpage layout.

Each sector can:

- rearrange composition
- alter node density
- shift atmosphere
- modify panel placement
- change spatial pacing

But these must remain consistent:

- node logic
- traversal rules
- environmental coherence
- system hierarchy

---

# Persistent Interface Layer

The following systems should exist nearly everywhere:

## Integrated Left Rail

The left rail should feel:

- embedded into the environment
- part station signage
- part system rail
- part pullout console

It should not feel like:

- a normal SaaS sidebar
- admin dashboard navigation

Possible contents:

- current sector
- route indicators
- operator message
- sound controls
- minimal nav
- Grid status

---

## Mini Node Map

The minimap is one of the most important persistent systems.

Behavior:

- transparent
- small
- corner-positioned
- clickable
- unlabeled
- exploratory
- MOBA-inspired

The minimap should:

- teach spatial memory
- imply scale
- encourage curiosity
- help orientation

without becoming overwhelming.

---

## Operator Messages

Operator messages should appear occasionally.

They can:

- guide
- tease
- humanize
- joke
- warn
- acknowledge events

Messages should feel:

- restrained
- observational
- procedural
- human

---

# Main Viewport Philosophy

The main scene should vary by sector.

This preserves the feeling that the user is moving through different environments rather than visiting different website pages.

Examples:

- homepage: station platform and route signage
- Lab: operator background, node clusters, dossier table
- project details: zoomed dossier surface or operational record
- deep sectors: stranger, denser, more atmospheric layouts

---

# Info Panel Philosophy

Information should be integrated into the scene.

Preferred behavior:

- camera zooms toward a dossier surface/tablet
- node interaction reveals an in-world detail surface
- panels feel physically placed in the environment

Avoid:

- detached modal windows
- generic slide-out web panels
- random floating SaaS cards

---

# Mobile Fallback Philosophy

Mobile should ideally be fully responsive with reduced motion.

If the full experience is not feasible, mobile should become a vertical story version that preserves:

- mood
- traversal idea
- project access
- core identity

Mobile can be less immersive, but it should not feel like an unrelated site.

---

# Wireframe Failure Conditions

A wireframe is wrong if:

- exploration flow feels off
- The Lab connects to unrelated ideas randomly
- traversal feels like normal page navigation
- the world feels too simple
- node relationships feel arbitrary
- information overwhelms atmosphere
- atmosphere blocks usability

---

# Final Wireframe Principle

The Grid’s wireframes should prioritize:

> exploration flow first, visual spectacle second.
